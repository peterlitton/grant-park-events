## 🐛 Critical Bug Fix

### **Issue:**
GSC functions (Request Index, Midnight Batch) failed with:
```
Error: error:1E08010C:DECODER routines::unsupported
```

**Symptoms:**
- Clicking "Request Index" in Admin Index Report failed
- Midnight batch job unable to submit URLs to Google
- Admin Index Report page loaded but couldn't submit URLs

**Root Cause:**
Netlify environment variable `GSC_PRIVATE_KEY` contains literal `\n` characters (backslash-n as two characters), but Node.js `crypto.sign()` requires actual newline characters for proper PEM format parsing.

**Impact:** Complete failure of Google Indexing API integration

---

## ✅ What's Fixed

### **Files Modified (2):**

**1. `netlify/functions/gsc-request-index.js` (line 125)**

**Before:**
```javascript
const privateKey = Netlify.env.get('GSC_PRIVATE_KEY');
```

**After:**
```javascript
const privateKey = Netlify.env.get('GSC_PRIVATE_KEY').replace(/\\n/g, '\n');
```

**Why:** Converts literal `\n` strings to actual newline characters before passing to `crypto.sign()`

---

**2. `netlify/functions/gsc-midnight-batch.js` (line 67)**

const privateKey = Netlify.env.get('GSC_PRIVATE_KEY').replace(/\\n/g, '\n');

## 🔬 Technical Analysis

### **Problem Validation:**
Comprehensive testing reproduced the exact error:

**Test Results:**
| Key Format | Works? | Error |
|------------|--------|-------|
| Actual newlines | ✅ Yes | None |
| Literal `\n` strings | ❌ No | `error:1E08010C:DECODER routines::unsupported` |
| After `.replace()` | ✅ Yes | None |

**Conclusion:** The `.replace()` transformation fixes the issue with 100% certainty.

### **Why This Happened:**
1. Netlify UI stores environment variables exactly as pasted
2. User pasted key with literal `\n` characters
3. Node's `crypto` module requires actual newline bytes for PEM format
4. OpenSSL DECODER routine couldn't parse the malformed PEM
5. Error propagated as DECODER unsupported error
