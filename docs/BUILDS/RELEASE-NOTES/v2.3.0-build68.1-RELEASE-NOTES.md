## 🐛 Bug Fix

### **Issue:**
Build68 "Updated" column showed "—" for all pages (events and static pages).

**Root Cause:**
`gsc-index-status.js` function was not passing `updatedAt` and `createdAt` fields from events to the frontend, even though events had these fields after migration.

**Impact:** "Updated" column feature completely non-functional.

---

## ✅ What's Fixed

### **Files Modified:**

**1. `netlify/functions/gsc-index-status.js`**

Added `updatedAt` and `createdAt` to event page objects:

```javascript
pages.push({
  url: eventUrl,
  type: 'event',
  status: status,
  details: details,
  eventDate: evt.date,
  updatedAt: evt.updatedAt,      // ← Added
  createdAt: evt.createdAt,      // ← Added
  requestData: requestData[eventUrl] || null
});
```

**2. `admin-index-report.html`**

Display format changes:
- **Column header:** "Last Edited" → "Updated" (shorter, prevents wrapping)
- **Display format:** "3d ago" → "01/28/26" (MM/DD/YY, matches Event Date format)
- **Tooltip:** Still shows full timestamp + time in CST on hover
