# Build62 Release Notes
**Grant Park Events - Google Indexing API Integration**  
**Version:** v2.3.0-build62  
**Date:** January 30, 2026  
**Build Type:** Major Feature Release

---

## 🎯 Overview

Build62 implements **full Google Indexing API integration**, enabling automated submission of URLs to Google for crawling. This replaces the previous manual Google Search Console workflow and saves 20+ hours per season through automation.

---

## ✨ New Features

### **1. Real Google Indexing API Integration**
- **File:** `netlify/functions/gsc-request-index.js`
- **What:** Direct submission to Google Indexing API (not just timestamp tracking)
- **Features:**
  - JWT-based authentication with service account
  - Automatic 3-attempt retry logic with exponential backoff (0s, 5s, 15s)
  - Error handling for: quota exceeded, invalid credentials, 404, network timeout
  - Saves request status and errors to Netlify Blobs for display

### **2. Enhanced Status Tracking**
- **File:** `netlify/functions/gsc-index-status.js`
- **What:** Returns full request data (not just timestamps)
- **Includes:** Request time, status, errors, attempt count, Google responses

### **3. Midnight Batch Job for Past Events**
- **File:** `netlify/functions/gsc-midnight-batch.js`
- **What:** Scheduled function runs daily at midnight CST
- **Purpose:** Auto-requests indexing for events 7+ days old
- **Result:** Forces Google to crawl and see noindex tag, removes from search faster
- **Schedule:** Runs at 6am UTC (midnight CST) using Netlify cron

### **4. Updated Admin Index Report UI**
- **File:** `admin-index-report.html`
- **Changes:**
  - Removed "Details" column (saves space)
  - Added tooltips/rollovers on Status column showing:
    - Event details (e.g., "Published, awaiting index")
    - Last checked timestamp (e.g., "2:45 PM today")
  - Error display in rollover format:
    - Shows error message
    - Shows attempt count
    - Format: "❌ Failed (3 attempts)" with hover tooltip
  - "Refresh Status" button already present (global refresh)

### **5. Google API Testing Tools**
- **File:** `admin-function-tests.html`
- **Added 6 new tests:**
  1. Google API Connection Test
  2. Authentication Test (credentials validity)
  3. Single URL Submission Test
  4. Status Check Test
  5. Error Handling Test
  6. Export Test Report (downloads JSON)
- All tests added to "Run All Tests" suite

## 🔧 Technical Implementation

### **Authentication Flow**
```
1. Load GSC_CREDENTIALS from Netlify env vars
2. Generate JWT with service account key
3. Exchange JWT for Google OAuth access token
4. Use access token for API calls (valid 1 hour)
```

### **Retry Logic**
```
Attempt 1: Immediate (0s delay)
Attempt 2: After 5s delay
Attempt 3: After 15s delay
Total max time: ~20s per URL
```

### **Error Types Handled**
- **Network timeout:** Retry up to 3 times
- **Already in queue:** Mark as success (not an error)
- **Quota exceeded:** Don't retry, save error
- **Invalid credentials:** Don't retry, save error
- **404 Page not found:** Don't retry, save error

### **Data Storage (Netlify Blobs)**
```json
{
  "url": "https://www.grantparkevents.com/events/...",
  "requestedAt": "2026-01-30T20:45:00Z",
  "status": "submitted|failed|already_queued|noindex_requested",
  "error": "Network timeout" (if failed),
  "attempts": 3,
  "googleResponse": "URL_UPDATED",
  "batchJob": true (if from midnight batch)
}
```

- **Total:** 40-70 requests/day (30-35% of quota)

## 🐛 Known Issues / Limitations

### **Current Limitations**
1. **No real-time Google status** - Index Report shows request status only (not actual Google index status)
   - Why: Google Search Console API has rate limits and doesn't provide real-time status
   - Workaround: Check status manually in Google Search Console if needed
   
2. **Midnight batch runs once per day** - Past events only processed at midnight
   - Why: Netlify free tier scheduling limitation
   - Workaround: Manual "Request Index" always available

3. **200 requests/day quota** - Google API hard limit
   - Why: Google's quota (not our limitation)
   - Impact: Not expected to hit limit based on usage patterns

### **Future Enhancements (Not in Build62)**
- Real Google index status checking (requires separate API calls)
- Bulk "Request Index" for multiple URLs at once
- Historical quota usage tracking
- Configurable midnight batch schedule

## 🆘 Troubleshooting

### **"GSC_CREDENTIALS not configured"**
- **Cause:** Environment variable not set in Netlify
- **Fix:** Add GSC_CREDENTIALS to Netlify env vars, redeploy

### **"Invalid credentials"**
- **Cause:** Service account not authorized or credentials malformed
- **Fix:** Verify service account added to Search Console, check JSON format

### **"Quota exceeded"**
- **Cause:** Hit 200 requests/day limit
- **Fix:** Wait until next day, quota resets at midnight UTC

### **"Network timeout"**
- **Cause:** Google API temporarily unavailable
- **Fix:** Retry automatically happens (3 attempts), or retry manually

### **Midnight batch not running**
- **Cause:** Netlify scheduled functions require manual enable
- **Fix:** Check Netlify dashboard → Functions → Scheduled, verify enabled
